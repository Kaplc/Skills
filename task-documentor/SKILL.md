# Task-Documentor - 开发任务文档生成器

**核心目标**: 将结构化的实现思路文档拆解为可执行、可量化的开发任务

**输入**: Idea-Documentor 生成的 `.md` 格式实现思路文档
**输出**: 按模块分组的开发任务文档，每个模块一个文件

---

## 逻辑链路（四阶段）

```
思路文档 → [解析结构] → [识别模块] → [拆解任务] → [生成文档]
```

---

## Stage 1: 解析结构

**目的**: 从思路文档提取关键信息

**输入**: 文档路径
**产出**: 结构化数据对象

| 提取项 | 来源位置 |
|--------|----------|
| 模块划分 | 4.4 模块划分表格 |
| 实现步骤 | 4.3 实现步骤表格 |
| 数据结构 | 4.5 数据结构设计代码块 |
| 核心机制 | 6.2 核心机制原理 |
| 优先级 | 9.1 优先级建议 |

---

## Stage 2: 识别模块与依赖

**目的**: 确定模块分类和执行顺序

**产出**: 排序后的模块列表

**分类规则**:
- **核心层**: 无依赖，基础数据结构
- **逻辑层**: 依赖核心层，核心算法
- **功能层**: 依赖逻辑层，具体功能
- **集成层**: 依赖多个模块，整合工作

**排序规则**: 被依赖的模块排在前面

---

## Stage 3: 任务拆解

**目的**: 将每个模块拆解为可执行的小任务

**拆解维度**:

| 操作类型 | 拆解方式 |
|----------|----------|
| 创建类/文件 | 一个类 = 一个任务 |
| 添加方法 | 一个方法 = 一个任务 |
| 添加字段 | 可合并多个相关字段为一个任务 |
| 实现接口 | 每个接口方法 = 一个任务 |
| 配置关联 | 每个关联 = 一个任务 |

**任务标准**:
- 原子化: 只做一件事
- 可执行: 含具体代码步骤
- 可验证: 有验收标准
- 时间可控: 2-5 分钟

**任务模板**:
```markdown
### [T-XXX] 任务名称

**所属模块**: {模块}
**预计时间**: {X}分钟
**任务类型**: {创建/修改/配置/集成}

**实现目标**: 一句话描述要实现什么

**实现思路**:
1. 步骤1
2. 步骤2
3. 步骤3

**验收标准**:
- [ ] 验收点1
- [ ] 验收点2

**依赖**: {T-XXX}
```

---

## Stage 4: 生成文档

**目的**: 输出符合规范的文档

**输出结构**:
```
docs/开发任务/{模块名}/
├── 开发任务_{模块名}_{日期}.md  (模块任务文档)
└── 开发任务总览_{日期}.md        (索引文档)
```

**模块任务文档结构**: 见 `TEMPLATE.md`
**索引文档结构**: 见 `INDEX_TEMPLATE.md`
